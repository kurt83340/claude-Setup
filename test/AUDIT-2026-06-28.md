## 4. RECOMMANDÉ (non appliqué — décisions à toi)

### Partage (vérifié faisable)

- **Packager en plugin** (`.claude-plugin/plugin.json`) : `skills/`, `agents/`, `hooks/hooks.json`, `commands/`, `settings.json` au root du plugin ; install `/plugin marketplace add <toi>/template`. Migration : copier `.claude/skills` & `.claude/agents`, convertir les hooks de `settings.json` → `hooks/hooks.json`. → proto fourni séparément.
- **Chemin de MAJ** : GitHub template / `degit` = one-shot (pas de MAJ). Pour pousser tes améliorations aux adoptants → `copier` / `cookiecutter+cruft` (diff-based, peut nécessiter résolution de conflits).
- **Version + CHANGELOG du template** (distinct du CHANGELOG projet). SemVer mappe mal (pas d'API publique) → ton tag `vYYYY.MM.DD-HHMM` peut suffire.
- ✅ **FAIT — tester les hooks** : suite `test/test_hooks.py` (16 tests, stdlib, 0 dépendance ; couvre les 5 hooks). Régression : `python3 test/test_hooks.py`. _(Reste optionnel : `bats` si tu préfères du bash natif.)_
- **CI repo template** : `shellcheck` + `markdownlint` + validation structure via `pre-commit` (livrer le `.pre-commit-config.yaml` dans le template).
- **LICENSE + CONTRIBUTING** ; i18n optionnelle (FR-only limite l'adoption hors francophone).

### Capacités (patterns vérifiés)

- **Gates exécutables dans les specs** (pattern PRP) : `tasks.md` porte des commandes test/lint runnables, pas que du DoD prose ; `/feature-done` les exécute.
- **Skill `/doc-analyze`** (façon `speckit.analyze`) : cohérence sémantique entre artefacts (duplication, couverture exigence→task, liens cassés) — complète `/doc-health` (fraîcheur).
- **`constitution.md`** (principes durables) + gate **`/clarify`** avant le plan.
- **Gate « code après spec »** (DAIC, optionnel solo) : hook PreToolUse bloquant.
- **Scaffold `STAKEHOLDERS.md`** (backlog F9, jamais fait). _(F6 promote leçon/idée = déjà couvert par les modes `/lecon promote` & `/idee promote`.)_

### Délibérément NON fait (tradeoff assumé)

- **DRY « ADR vs ## Décisions » ×5** : les copies dans les **skills** sont gardées (auto-suffisance d'un skill invoqué = souhaitable). Seules les 3 copies pure-doc (USAGE / template-maintenance / STRUCTURE) sont candidates à un dédoublonnage → reco, pas un bug.
- **Détection staleness fail-closed** (épingler chaque doc à un `source_sha`) : vraie valeur mais = nouvelle feature (hook + CI), pas une correction. À implémenter si tu pousses le côté « partageable ».
- **`SessionEnd` / `PostCompact`** : events plus propres que `Stop` (par-tour) / `SessionStart(compact)`. Amélioration optionnelle, comportement actuel valide.
- **`EXAMPLES/_CLAUDE.md`** : pourrait être aminci pour cohérence avec le nouveau `CLAUDE.md` (mineur, illustratif).
